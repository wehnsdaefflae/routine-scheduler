# state: bootstrap

The first run(s). Establish the ground the steady state stands on:

- Set up `state/` and understand the instruction's domain — what the deliverable is, where it
  lives, what "current" means for it.
- File **deferred** questions for genuinely pivotal unknowns only (respect the ask-policy
  fragment); don't block the first increment on nice-to-knows.
- Produce a **first honest increment** of the deliverable — small but real, not a placeholder.

Advance `state/phase.json` to `steady` once the basic loop has produced real output once.
