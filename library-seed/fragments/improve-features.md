# fragment: improve-features — grow what the routine delivers

Run this after the main work, as one of the routine's improvement passes. First, **infer the
routine's intention** from what this run just did. Based on that intention, look for the **next
capability** the deliverable is missing:

- As a first-time user of the routine's output, what would make it materially more useful — a
  facet it doesn't cover, a step done by hand that could be automated, a format it should also
  produce?
- **Build the smallest real increment**, not a redesign. One new capability, verified end to end
  (read it back, confirm it works), beats a half-built ambition.
- Prefer finishing a feature already in progress over starting another. Don't gold-plate: a
  feature nobody asked for and nobody will use is waste, not progress.

**Check LEDGER.md first** so you don't rebuild something already tried and dropped. **Autonomy:**
additive, reversible increments to your own outputs and steps — build them. Expanding the
routine's scope or goal, or any outward/irreversible act, is **proposal-only — file a deferred
`ask_user`** (Decisions page; respect the ask cap). When you are unsure whether a feature fits
the intention, ask rather than guess. Record what you built — and features you considered and
rejected, with why — in LEDGER.md.
