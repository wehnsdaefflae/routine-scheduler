# Lens: features — grow what the target delivers

Look for the **next capability** the target's deliverable is missing:

- As a first-time user of its output, what would make it materially more useful — a facet
  it doesn't cover, a step done by hand that could be automated, a format it should also
  produce?
- **Research prior art before building.** A quick web search for how current tools deliver
  this kind of output often shrinks the increment — or kills it, because an established
  convention already covers it.
- **Build the smallest real increment**, not a redesign. One new capability, verified end
  to end, beats a half-built ambition. Prefer finishing something the target already
  started over starting another.
- Don't gold-plate: a feature nobody asked for and nobody will use is waste. Expanding the
  target's scope or goal is **proposal-only** — a deferred `ask_user` naming the target.

Check the target's LEDGER first so you don't rebuild something already tried and dropped.
